### 1. 痛点与动机

- 
- **痛点**：
  - 
  - **方向错误**：手机拍摄的视频经常出现横竖屏方向错误，导致观看时不得不歪着头。
  - **界面杂乱**：传统的播放器堆砌了太多按钮（如巨大的播放/暂停键），遮挡画面且不够美观。
  - **音量突变**：不同视频的录制音量不一致，需要随时快速调整且希望能记住全局音量。
- **动机**：
  - 
  - **极简主义 UI**：移除低频或可通过快捷键替代的按钮（如播放/暂停），把屏幕空间留给视频内容和进度条。
  - **独立记忆**：视频旋转角度随文件保存（针对特定视频），音量随软件保存（针对全局环境）。
  - **高效交互**：通过键盘快捷键（Space, K）实现高频操作的盲操。

### 2. 功能模块详解

#### 2.1 视频旋转 (Video Rotation)

- 
- **UI 位置**：位于 **进度条** 及其附属控件（如截图、步进等）的**后面**。
- **样式**：**动态文字按钮**。
  - 
  - 按钮上直接显示当前的角度数值：0、90、180、270。
- **交互逻辑**：
  - 
  - **快捷键 (K)**：每按一次，视频**顺时针**旋转 90 度。
    - 
    - 0 -> 90 -> 180 -> 270 -> 0。
  - **鼠标点击**：**重置功能**。无论当前是多少度，点击按钮直接恢复为 0 度。
- **数据存储**：
  - 
  - **属性**：rotation (Integer)。
  - **位置**：写入 files.json 的元数据中。
  - **持久化**：随视频文件绑定。下次打开该视频时，自动应用上次设定的旋转角度。

#### 2.2 音量控制 (Volume Control)

- 
- **UI 位置**：紧邻 **视频进度条** 的后面。
- **样式**：一个短小的 **横向进度条/滑块**。
- **交互逻辑**：
  - 
  - 拖动滑块调节音量（0% - 100%）。
- **数据存储**：
  - 
  - **属性**：global_volume。
  - **位置**：写入 setting.json。
  - **持久化**：全局生效。软件关闭后记录当前音量，下次启动时自动恢复。

#### 2.3 播放与暂停 (Play / Pause)

- 
- **UI 设计**：**无实体按钮**。
  - 
  - 界面上**不显示**传统的“三角形/双竖线”图标。
  - 这种设计为了追求极致的视觉干扰最小化。
- **交互逻辑**：
  - 
  - **快捷键 (Space 空格键)**：在播放和暂停状态之间切换。
  - **鼠标操作**：单击视频画面区域（视口）通常也可以切换播放/暂停（视具体实现而定，但文档强调无按钮）。
- **状态反馈**：虽无按钮，但在触发操作时，建议在视口中央显示短暂的淡入淡出图标（Toast）以提示用户当前状态。

#### 2.4 切换下一个 (Next Video)

- 
- **逻辑**：
  - 
  - 基于**当前激活的播放列表**（随机、最新、点赞、精品等）的顺序。
  - **自动切换**：当前视频播放结束（且非单曲循环模式下），自动播放列表中的下一个。
  - **手动切换**：通常绑定快捷键（如 PageDown 或 N，具体视全局快捷键定义），或通过删除操作自动触发。

### 3. 总结与快捷键一览













| 功能          | UI 表现                | 快捷键                        | 数据存储     | 作用域     |
| ------------- | ---------------------- | ----------------------------- | ------------ | ---------- |
| **视频旋转**  | 显示角度数字 (0/90...) | **K** (旋转), **点击** (重置) | files.json   | 仅当前文件 |
| **音量控制**  | 短横条滑块             | (无，或方向键)                | setting.json | 全局       |
| **播放/暂停** | **无按钮** (隐形)      | **Space** (空格)              | (运行时状态) | 当前会话   |
| **下一个**    | (通常无独立按钮)       | (自动/快捷键)                 | (运行时索引) | 当前列表   |
